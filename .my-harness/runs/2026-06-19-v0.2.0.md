## 关键决策

- D1：范围选择真实 IAM 验收 + 一个第三方应用接入包。
- D1（Step 2）：不新增配置 UI。
- Step 3：无新增主要 UI，沿用 `DESIGN.md`、`design/base-portal-visual-target.md`、`design/base-portal-final-design-option-2.md` 的 Light Command Workspace。
- Step 5：接入包使用 `config/portal-apps/*.json` + ops API；IAM sync 覆盖 permission point、permission group 和 binding；真实证据必须脱敏。
- OAuth 根路径回调修复：静态资源中间件禁用默认 `index`，`/?code&state` 必须交给 Nest controller 处理，避免真实 IAM callback 被 SPA shell 截获。
