# base-portal v0.3.0 my-harness Run

日期：2026-06-20

## 当前范围

v0.3.0 做第三方无感接入体验版。范围包含：

- `#11` SSO Demo 菜单嵌入页必须实现无感登录。
- `#12` 移除沉浸模式。
- `#13` Tab 右键菜单和 Tab 操作菜单增加刷新当前 iframe。

交付终点为 release + deploy，并完成线上健康检查和关键路径验证。

## 已确认决策

- Step 1：版本定位为 `v0.3.0` 第三方无感接入版，范围为 `#11/#12/#13`，交付到 release + deploy。
- Step 2 D1：SSO Demo 自己完成 OAuth/session，Base Portal 只负责入口编排、权限过滤、iframe/new-window/fallback 和可恢复体验，不传递 token、cookie、code 或 secret。
- Step 2 D2：移除沉浸模式的产品入口和运行态；历史 `openMode=immersive_iframe` 运行时兼容降级为标准 iframe，不做高风险 schema migration。
- Step 2 D3：Tab 右键菜单和 Tab 三点菜单都提供刷新当前 iframe，底层共用同一个只刷新当前业务 Tab iframe 的动作。
- Step 3：不新增完整 Product Design 视觉稿或 Pencil 原型；沿用 `Light Command Workspace`，通过轻量设计决策文档和 `DESIGN.md` 同步收口。
- Step 4：Product Design review 通过；设计制品足以进入工程评审，不需要新增完整视觉稿。
- Step 5：Engineering review 通过；采用 `new_tab` 作为 hint、`immersive_iframe` 运行态移除并兼容降级、iframe refresh revision 状态模型、SSO Demo 配合需求文档。
- Step 6：Superpowers `writing-plans` 完成；根入口 `IMPLEMENTATION_PLAN.md` 指向规范计划 `docs/superpowers/plans/2026-06-20-v0.3.0-embedded-sso-workspace.md`。
- Step 7：实现完成；版本默认值、workspace refresh、移除沉浸入口、接入包兼容、seed 和 SSO Demo 菜单包已落地。
- Step 8：本地工程验证通过；`pnpm check`、`pnpm build`、Compose config、脚本语法和 `git diff --check` 均通过。
- Step 9：浏览器验证通过；桌面、平板、手机覆盖 SSO Demo iframe、右键刷新、三点菜单刷新、无沉浸入口和最大化。
- Step 10：Visual QA 通过；无需新增完整视觉稿。
- Step 11：Functional QA 通过；v0.3.0 范围内无阻塞问题。
- Step 12：Pre-landing review 通过；未发现阻塞 ship / deploy 的代码问题。
