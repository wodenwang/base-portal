## Step 5 handoff

下一步：my-harness Step 5 `gstack /plan-eng-review`。

工程评审必须重点挑战：

- 真实 OAuth code -> token -> userinfo -> permissions 链路的错误处理和脱敏验证方式。
- 应用接入包配置格式、校验、幂等 upsert 和误删防护。
- `/api/ops/sync-iam-resources` 的 dry-run/apply 模型、permission points、permission groups、group-point binding 和失败摘要。
- IAM developer API 的权限边界，确保不修改应用配置、登录凭证、角色授权或管理员授权。
- `PortalDomain` / `PortalMenu` schema 是否足够，是否需要 migration。
- `PortalAuditEvent` 是否足够记录配置导入、sync、登录和菜单访问。
- 内存 session 在生产重启时丢失是否接受，或是否必须在 v0.2.0 引入持久 session。
- 真实生产验收的非敏感证据规则，避免泄露 authorization code、access token、cookie、client secret、developer token。
- `DEPLOY.md` 对版本、镜像、install/upgrade、健康检查和线上读回的要求。
