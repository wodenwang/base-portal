# Base Portal 视觉目标

日期：2026-06-19

## Product Design Brief

Base Portal 是一个企业内部门户壳，用于承载多个独立第三方系统。首版视觉目标是“运营工作台感”：比传统后台更活跃、更有效率工具气质，但不做通知、待办、搜索、个性化工作台等聚合功能。

视觉目标服务这些核心界面：

- 顶部品牌和功能域切换。
- 左侧最多三层菜单。
- Tab 工作区。
- 固定功能域首页 tab。
- iframe 嵌入容器。
- 应用内最大化。
- iframe fallback 面板。
- 桌面和平板优先的响应式布局。
- v0.3.0 当前 iframe 刷新和 SSO Demo 可恢复访问。

## 视觉原则

- 清晰优先：门户壳必须降低认知负担，帮助用户快速定位功能域、菜单和当前打开页面。
- 统一但不压迫：Portal 视觉要有品牌感，但不能干扰第三方系统页面。
- 工作区气质：Tab、菜单、功能域切换应该像高频办公工具，操作密度适中，状态清楚。
- 少卡片化：主壳层和页面区域不堆叠卡片，固定首页可以使用轻量信息区域，不做营销式 hero。
- 桌面/平板优先：默认宽屏布局精致，窄屏变为可用兜底。

## 布局目标

顶部：

- 左侧展示 `Base Portal` 品牌。
- 中间展示功能域一级菜单，支持溢出。
- 右侧展示头像/姓名和退出登录。

左侧：

- 展开态显示三层菜单。
- 收起态保留图标和 tooltip。
- 窄屏通过菜单按钮或抽屉打开。

工作区：

- Tab 条承载固定首页和业务 tab。
- 固定首页 tab 不计入 20 个业务 tab 上限。
- 当前 tab 状态要明确。
- 关闭、关闭全部、关闭其他动作放在稳定位置，避免误触。

嵌入容器：

- iframe 模式保留完整门户壳。
- 最大化是临时覆盖态，隐藏顶部、左侧和 tab，退出后回到标准 iframe 工作区。
- v0.3.0 移除沉浸模式；历史 `immersive_iframe` 数据运行时兼容降级为标准 iframe。
- 业务 tab 必须提供刷新当前 iframe 的恢复动作。

## 状态目标

必须覆盖：

- 未登录。
- 登录中。
- 已登录空权限。
- 功能域首页。
- 菜单打开。
- tab 达到上限。
- 含 `confirm_on_close` 的关闭确认。
- iframe fallback。
- 手机兜底菜单。

## 组件方向

实现阶段优先使用：

- `lucide-react` 图标。
- Tailwind CSS 设计变量。
- Radix UI 的 dropdown、dialog、tooltip、alert-dialog 等基础交互。
- 自定义 Workspace、TabStrip、DomainNav、SidebarTree、EmbedFrame 组件。

## 需要后续实现验证的点

- 1440px 桌面视口下，功能域、左侧菜单、tab 和 iframe 不拥挤。
- 834px 平板横向或接近宽度下，菜单和 tab 仍可操作。
- 手机宽度下，Portal 壳不破版，菜单入口、tab 管理和退出最大化可用。
- iframe fallback 面板不能遮挡用户可以复制或新窗口打开的操作。
