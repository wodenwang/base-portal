## Product / interaction review

### 1. v0.2.0 的用户价值成立

`v0.1.2` 证明门户壳和部署链路可运行，但还没有证明它能承载真实 IAM 用户和真实第三方应用接入。v0.2.0 应优先补齐这个缺口。

推荐产品承诺：

- 运维者能用无 secret 的应用接入包声明一个第三方系统。
- 管理员能 dry-run 和 apply 权限资源同步，并看到非敏感摘要。
- 真实 IAM 用户登录后，只看到自己有权限的菜单。
- 生产验收能证明 OAuth、userinfo、permissions、导航过滤和审计闭环。

这比单纯补 OAuth smoke 更有产品价值，也比完整管理后台更符合当前边界。

### 2. 不建议在 v0.2.0 做完整配置 UI

`DESIGN.md` 已明确当前不是完整 Admin Console / CRUD 后台，Discovery 也确认“管理后台先弱化”。因此 v0.2.0 不应新增菜单管理、权限绑定管理、应用编辑器或多步骤表单。

推荐交互形态：

- `config/portal-apps/*.json` 或等价版本化配置文件作为源输入。
- ops API 提供 `dryRun`、`apply`、校验错误和审计记录。
- CLI / curl / 文档记录作为运维入口。
- 前端只验证用户可见路径：登录、空权限、部分权限、菜单进入、iframe/new tab/immersive 入口。

若后续确实需要 UI，放到 v0.3.0 或单独版本，先走新的 Step 3 视觉目标。

### 3. 真实 IAM 验收需要“非敏感证据模式”

真实 OAuth 验收不能把 authorization code、access token、cookie、client secret 或 developer token 写入日志、截图、文档或聊天。

推荐证据形态：

- 登录入口点击和回跳只记录 URL origin/path、HTTP status、页面状态，不记录 query 中的 code。
- session 验证只记录 `authenticated=true`、脱敏 user id 或稳定 hash、permission point 数量和 key 前缀。
- 权限读取只记录权限 key 列表或数量，不记录 token。
- 截图只展示 Portal 壳、用户名可按需要脱敏，不展示 cookie/devtools token。
- 失败记录只显示错误码、阶段和 request id，不输出响应体中的敏感字段。

### 4. dry-run/apply 的反馈应该结构化，但不一定需要 UI

当前 `/api/ops/sync-iam-resources` 已有 permission point dry-run/apply 基础，但 v0.2.0 需要更明确的反馈模型：

- `dryRun=true`：只返回 wouldCreate/wouldUpdate/wouldBind/warnings/errors。
- `apply=true`：返回 created/updated/skipped/conflicted/bound/failed 摘要。
- 每个条目只展示 key、name、状态、错误码；不展示 token 或 secret。
- 审计记录保存 actor、count、dryRun/apply、result status、配置包版本或 hash。

这类反馈先用于 API 和文档即可。若后续要放 UI，可映射为 shadcn `Alert`、`Table`、`Badge`、`Dialog`，但本版本不需要先做。

### 5. 前端现有状态基本够用，但验收要覆盖三类权限结果

v0.2.0 不需要重做门户壳视觉。应沿用当前状态页并补验收：

- 无权限：已登录但没有可访问菜单，展示空权限状态。
- 部分权限：只显示一个接入包中授权的叶子菜单，父级自动保留。
- 全权限：显示该接入包全部可访问菜单。

若发现现有文案不足，可做小文案和状态改动，但不需要新视觉稿。

## Risks to challenge in engineering review

- session 当前是内存 Map，生产容器重启会丢会话；v0.2.0 是否接受这个限制需要工程评审明确。
- permission groups 和 group-point binding 是否纳入本版本，会改变 IAM sync 的幂等和失败处理复杂度。
- 应用接入包是否允许删除/停用菜单，需要避免误删生产菜单。
- 第三方系统 iframe 是否可嵌入受 CSP / X-Frame-Options 约束；Portal 只能提供 fallback，不能保证第三方页面可嵌。
- 真实 IAM 验收若依赖生产账号，需要先定义谁授权、谁执行、如何脱敏证据。

| 选项 | 推荐 | Pros | Cons | 影响范围 |
