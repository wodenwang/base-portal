# Base Portal v0.3.1 范围决策

日期：2026-06-27

## 结论

v0.3.1 是一个部署治理和范围收口补丁版本，不新增前端页面。

本版本明确下架权限相关 UI：

- 不做权限矩阵。
- 不做权限同步状态页。
- 不做审计入口。
- 不新增权限绑定编辑、权限组选择器、权限点表格或批量操作界面。
- 不新增 Product Design 视觉稿或 Pencil 原型。

## 采用的范围

- 修正 `deploy/docker-compose.yml` 中 `BASE_PORTAL_VERSION` 和 `APP_VERSION` fallback 仍停在旧版本的问题。
- 将仓库版本、部署默认值、安装脚本、升级脚本、README、DEPLOY 和 release notes 同步到 `v0.3.1`。
- 保留 v0.3.0 已交付的 SSO Demo 菜单、iframe refresh、接入包导入、IAM sync 和权限过滤能力。
- 生产部署后读回 `.deploy/version`、容器 image、`/health`、`/ready`、`/version` 和公网登录页。

## 不做事项

- 不改数据库 schema。
- 不改 Feishu IAM OAuth、permission API 或 developer API 调用边界。
- 不改第三方接入包 JSON 结构。
- 不新增管理后台或完整 CRUD 配置 UI。
- 不读取、回显或提交真实 secret、token、cookie、authorization code、数据库密码或证书私钥。

## 设计判断

由于本版本不新增用户可见权限 UI，`DESIGN.md` 和既有 `Light Command Workspace` 视觉目标继续有效，不需要新的 Product Design 三方案选型或完整视觉评审。

后续若恢复权限矩阵，应作为新版本重新进入 my-harness 流程，并从 Product Design `get-context` 和 `ideate` 开始产出可选视觉方向。
