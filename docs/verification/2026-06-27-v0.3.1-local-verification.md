## 结论

本地工程验证通过。v0.3.1 本地门禁覆盖：

- 版本号、部署示例、安装脚本和升级脚本已更新到 `0.3.1` / `v0.3.1`。

## 环境说明

当前 Codex runtime 的默认 `pnpm` 是 `11.7.0`，仓库 `packageManager` 声明为 `pnpm@9.15.4`。

`pnpm@11` 会触发 `approve-builds` / `allowBuilds` 新策略并污染 `pnpm-workspace.yaml`，因此本次校验使用以下 PATH 强制裸 `pnpm` 嵌套调用也走仓库声明版本：

```bash
PATH=/Users/wenzhewang/.nvm/versions/node/v24.16.0/bin:$PATH
```

首次安装后需要生成 Prisma Client：

```bash
PATH=/Users/wenzhewang/.nvm/versions/node/v24.16.0/bin:$PATH CI=true pnpm --filter @base-portal/api prisma:generate
```

结果：通过。Prisma Client v6.19.3 已生成。


结果：全部通过。

```bash
docker buildx build --platform linux/amd64 -f deploy/web.Dockerfile -t base-portal-release:v0.3.1 --load .
```

首次结果：失败。失败原因为仓库缺少 `.dockerignore`，Docker build context 把本机 `apps/*/node_modules` 一起复制进镜像，污染容器内 pnpm 工作区链接，导致容器内 `nest build` 解析不到 `@nestjs/core`。

修复：新增 `.dockerignore` 排除 `node_modules`、`apps/*/node_modules`、`apps/*/dist`、`deploy/.env`、`deploy/data`、输出目录和日志。

重新运行结果：通过。镜像成功导出为 `base-portal-release:v0.3.1`，容器内 `prisma:generate`、`apps/api` Nest build 和 `apps/portal-web` Vite production build 均通过。
