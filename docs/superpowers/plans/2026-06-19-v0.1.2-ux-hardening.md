# Base Portal v0.1.2 UX Hardening Plan

This is the Superpowers execution plan for `base-portal` v0.1.2. The project-facing copy is mirrored in `IMPLEMENTATION_PLAN.md`.

## Objective

Ship and deploy v0.1.2 with UX hardening for GitHub issues #6-#9:

- #6: keep business tab close buttons fixed on the right.
- #7: support business tab reorder through desktop drag/drop plus explicit move fallback.
- #8: redesign collapsed sidebar as Compact flyout sidebar.
- #9: normalize ContextMenu/DropdownMenu pointer, hover, focus and disabled states.

## Constraints

- Do not change Feishu IAM auth, permission, audit, database schema or deploy topology.
- Do not add a second UI framework.
- Do not introduce tab persistence, menu management UI or broader workbench features.
- Use a pure workspace model function for tab reordering and route UI drag/drop/fallback through that model.
- Keep home tab fixed and non-draggable.
- Mobile does not need touch drag; it must remain usable through menus and stable layout.

## Implementation Checklist

- [ ] Add `reorderTabs` to `apps/portal-web/src/workspace.ts`.
- [ ] Cover reorder behavior in `apps/portal-web/src/workspace.test.ts`.
- [ ] Wire desktop drag/drop and move fallback in `apps/portal-web/src/App.tsx`.
- [ ] Implement fixed tab title/close layout in `apps/portal-web/src/styles.css`.
- [ ] Implement collapsed sidebar flyout in existing `MenuTree` / `MenuNode`.
- [ ] Normalize Radix menu item, separator, highlighted and disabled styles.
- [ ] Bump v0.1.2 version defaults in package, health fallback, deploy examples and scripts.
- [ ] Run required local checks.
- [ ] Capture Playwright desktop/tablet/mobile evidence.
- [ ] Prepare release docs, commit, tag, GitHub release, remote deploy and production readback.
