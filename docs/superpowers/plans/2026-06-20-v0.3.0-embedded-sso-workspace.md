# Base Portal v0.3.0 Embedded SSO Workspace Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task. Update checkboxes as tasks complete.

**Goal:** Ship and deploy Base Portal `v0.3.0` as the third-party no-touch access experience release, covering GitHub `#11/#12/#13`: SSO Demo embedded entry with recoverable fallback, removal of immersive mode, and refresh-current-iframe actions in the tab controls.

**Architecture:** Keep the existing NestJS + React + Prisma + Docker deployment shape. Base Portal remains the portal shell and permission-filtered entry orchestrator; it does not proxy SSO Demo OAuth/session credentials. Workspace tabs use a standard iframe runtime only. Historical `immersive_iframe` data is normalized to `iframe` at runtime/import boundaries without a schema migration. Refresh is modeled as a per-tab iframe revision, so only the selected business iframe remounts.

**Tech Stack:** NestJS 10, Prisma, PostgreSQL, React/Vite, shadcn/ui, Radix primitives, lucide-react, Vitest, Playwright, Docker Compose, bash deploy scripts, Feishu IAM OAuth/developer APIs.

---

## Scope Decisions

- SSO Demo owns its own OAuth/session. Base Portal only provides menu entry, permission filtering, internal iframe tab, new-window fallback, and recovery affordances.
- Base Portal must never pass token, cookie, authorization code, client secret, developer API token, or other session secret to SSO Demo.
- `immersive_iframe` is removed from user-visible UI and frontend workspace runtime.
- Historical `openMode=immersive_iframe` is compatible-downgraded to standard `iframe`; no database schema migration in v0.3.0.
- `new_tab` remains a data/opening hint. Normal menu click still opens or activates an internal business tab; explicit new-window recovery is exposed from tab menus and fallback UI.
- Right-click tab context menu and tab three-dot menu call the same refresh-current-iframe action.
- No new complete Product Design visual or Pencil prototype is required; continue with `DESIGN.md`, `design/base-portal-visual-target.md`, `design/base-portal-final-design-option-2.md`, and `design/2026-06-20-v0.3.0-design-decision.md`.

## Non-Scope

- No config management UI.
- No Base Portal secondary authentication for SSO Demo.
- No Feishu IAM core protocol changes.
- No persistence of tab order or iframe refresh revisions.
- No database schema migration to delete `open_mode`.
- No reading or printing real env values, cookies, OAuth codes, access tokens, client secrets, developer API tokens, database passwords, or certificate private keys.
