# Base Portal v0.3.1 Permission UI Scope Closeout Plan

> **For agentic workers:** This is a small patch release plan. It intentionally skips new Product Design artifacts because the selected product decision is to ship no permission-related UI in this version.

**Goal:** Ship and deploy `v0.3.1` as a deployment-governance patch that removes permission UI from the current version scope and fixes version default drift in Docker Compose.

**Architecture:** Keep the existing NestJS + React + Prisma + Docker Compose shape. No schema migration, no new frontend route, and no new permission management UI. Existing ops APIs remain available for controlled operator use.

**Tech Stack:** Existing Base Portal stack: NestJS 10, Prisma, PostgreSQL, React/Vite, Vitest, Docker Compose, bash deploy scripts.

## Scope Decisions

- Permission matrix is out of scope for `v0.3.1`.
- Permission sync status UI is out of scope.
- Audit entry UI is out of scope.
- Product Design visual ideation is not required because the version adds no UI.
- Fix the `deploy/docker-compose.yml` fallback version drift so `/version` can reliably report `0.3.1` when env values are omitted.
- Release and deploy with fixed image tag `base-portal-release:v0.3.1`.
